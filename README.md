# astrbot_plugin_wakeup

**隐式生物钟与主动唤醒引擎** —— 让大模型自主规划下一次找你的时间。

## 它做什么

LLM 在回复末尾输出 `[NEXT: Xm]` → 插件静默截获 → 设定定时任务 → 倒计时结束后调用 LLM → LLM 决定是否主动发消息。

整个过程用户无感知，LLM 像有自己的生物钟一样自主运作。

## 快速上手

### 1. 安装

将 zip 上传到 AstrBot 插件管理页面，或通过插件市场安装。

### 2. 教 LLM 使用标签

在你的人格 prompt 中加入类似指引（根据角色设定调整措辞）：

```
你可以在回复末尾使用以下标签来规划自己的作息：
- [NEXT: Xm]：X 分钟后系统会提醒你，届时你可以决定是否主动联系用户

示例：
"晚安，明天见[NEXT: 480m]"       ← 8小时后被唤醒
"[NEXT: 30m]"            ← 30分钟后再决定要不要找她
"在忙 等我一下[NEXT: 10m]"       ← 10分钟后主动回来
```

### 3. 配置项说明

| 配置项 | 说明 | 默认值 |
|--------|------|--------|
| `persona_id` | 唤醒时加载的人格名称。留空自动查找 | 空 |
| `allowed_models` | 模型白名单，留空允许所有 | 空 |
| `trigger_keyword` | 截获的标签关键词 | `NEXT` |
| `wakeup_prompt` | 唤醒提示词，支持 `{{current_time}}` 占位符 | 内置默认 |
| `bubble_separator` | 消息分段符号，留空不分段 | 空 |
| `clean_patterns` | 发送前的清理正则列表 | `[NO_REPLY]` 和 `[NEXT]` |
| `default_silence_minutes` | 兜底休眠时间（分钟），设 0 关闭 | `360` |
| `timezone` | 时区，留空跟随 AstrBot 全局设置 | 空 |
| `max_retries` | 唤醒失败最大重试次数 | `5` |
| `retry_delay` | 重试间隔（秒） | `120` |

### 4. 指令

- `/wakeup` — 查看当前活跃的闹钟及剩余时间（不会砸碎闹钟）

## 工作原理

1. LLM 正常回复时输出 `[NEXT: 30m]`
2. `on_llm_response` 钩子截获标签，设定 30 分钟定时任务
3. 用户在此期间发消息 → 闹钟被砸碎，由新对话接管
4. 30 分钟到 → 加载人格 + 对话历史 → 裸调 LLM（附带当前时间）
5. LLM 生成回复 → 按分隔符拆分 → 逐条发送
6. 回复写入对话历史 → 检查新标签 → 设下一个闹钟
7. 循环往复，生物钟永不停歇

## 关键设计

- **人格加载**：配置 persona → 会话 persona → 全局默认人格，三级 fallback
- **持久化**：闹钟信息写入 JSON 文件，插件重载/AstrBot 重启后自动恢复
- **防冲突**：唤醒发消息期间标记 umo，防止自己触发的事件砸碎闹钟
- **闹钟安全**：finally 块通过比较 task 引用，不会误删新设的闹钟
- **砸碎中断**：多检查点中止机制，避免砸碎后仍发出消息
- **时间感知**：`{{current_time}}` 占位符在运行时替换为当前时间

## 已知限制

- **前台日志不显示唤醒消息**：`send_message()` 绕过了 AstrBot 响应 pipeline
- **唤醒时无法调用工具**：裸调 `text_chat()` 不走 AstrBot 工具链，MCP 工具不可用
- **对话历史格式**：部分 AstrBot 版本中后台可能显示为 JSON 字符串，用户发消息后恢复

## 环境要求

- AstrBot v4.22+
- 支持任意平台适配器（QQ/aiocqhttp、Telegram 等）
