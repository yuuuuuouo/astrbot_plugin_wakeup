## 🌟 这是什么？

这是一个为 [AstrBot](https://github.com/AstrBotDevs/AstrBot) 打造的 **"隐式生物钟"** 插件。

与市面上"定时推送"或"随机打扰"型插件不同，本插件的创新点在于：**下一次主动找你的时间，由大模型自己决定。**

模型在每次回复的末尾悄悄输出一个 `[NEXT: Xm]` 标签，系统底层静默截获并转化为定时任务。这个标签在聊天界面完全不可见，但它就是模型的"潜意识"——他打算多久之后再想起你。

| 传统定时插件 | 本插件 |
|:---|:---|
| 系统驱动，机械打扰 | 大模型潜意识驱动 |
| 不管情绪氛围，时间到就发 | 根据上下文、情绪、状态自主决定 |
| 冷战？热聊？一视同仁 | 生闷气定 24h，占有欲爆棚定 1min |

## ✨ 核心特性

- 🧠 **自主时间规划**：模型在回复中输出 `[NEXT: Xm]` 标签，由它自己决定何时再来
- 🕶️ **完全隐身**：标签在聊天界面不可见，通过消息过滤正则底层拦截
- 🔨 **防堆叠锁**：连续回复时自动砸碎旧闹钟，永远只保留最新的
- 🩺 **心跳兜底**：模型忘记打标签时自动兜底（可配置），生物钟永不停止
- ♻️ **重启恢复**：AstrBot 重启后自动恢复所有未到期的闹钟，不会失联
- 🎭 **模型白名单**：避免在昂贵模型或非闲聊模型下触发
- 🛡️ **防诈尸机制**：用户主动说话时立即砸碎倒计时，不会"打扰到自己"
- 🔁 **失败重试**：唤醒失败自动重试，默认 5 次 × 120s 间隔
- 💭 **优雅静默**：模型可以选择 `[NO_REPLY]` 保持静默，但心跳依然在跳

## 🔬 工作原理

```
┌──────────────────────────────────────────────────────────────┐
│                      消息流完整生命周期                        │
└──────────────────────────────────────────────────────────────┘

  用户发送消息
       │
       ├─→ on_user_message 钩子：砸碎当前闹钟（防诈尸）
       │
       ▼
  LLM 请求
       │
       ▼
  on_llm_response 钩子 ★
       │
       ├─ 读 resp.completion_text（LLM 原始输出）
       ├─ 正则匹配 [NEXT: Xm]
       │
       ├─→ 命中 → 设定 asyncio 闹钟任务（防堆叠 + 写入 JSON）
       │
       └─→ 未命中 → 启用兜底（default_silence_hours 小时后唤醒）
                                │
                                ▼
  消息过滤正则（WebUI 配置）在发送阶段把 [NEXT: Xm] 从文本里剥掉
                                │
                                ▼
  用户看到的：模型正常的回复，看不到任何标签
                                │
                                ▼
  ⏰ 倒计时结束
                                │
                                ├─→ 严出：当前模型不在白名单 → 取消
                                │
                                ▼
  用 wakeup_prompt 调 LLM（带会话历史）
                                │
                                ├─→ LLM 回复里含 [NO_REPLY] → 保持静默
                                │
                                └─→ 有内容 → 发送给用户
                                        │
                                        ▼
                              继续扫描回复里的 [NEXT: Xm]
                                （心跳不停，循环往复）


┌──────────────────────────────────────────────────────────────┐
│                       重启恢复流程                             │
└──────────────────────────────────────────────────────────────┘

  AstrBot 启动 → 插件加载 → initialize() 钩子触发
       │
       ▼
  读取 data/astrbot_plugin_wakeup/wakeup_alarms.json
       │
       ├─→ 剩余时间 > 0 → 按剩余秒数重建 asyncio 任务
       │
       └─→ 已过期 → 5s 后立即触发一次唤醒（补偿停机期间的失约）
```

### 🗝️ 三条设计原则

**宽进严出：**
- **宽进**：任何模型输出 `[NEXT: Xm]` 都先设闹钟，不做前置检查
- **严出**：闹钟触发时才检查当前模型是否在白名单。这样即使你中途切换到非白名单模型，也不会让别的模型误用 API

**心跳永不停：**
- 只要 LLM 响应过（无论是对话还是唤醒），就一定会产生下一次闹钟
- 模型可以用 `[NO_REPLY]` 选择静默，但"要不要被问醒"由系统兜底
- 这保证了即使模型连续偷懒不打标签，生物钟也不会彻底失联

**状态不丢：**
- 每次设定/取消闹钟都会即时持久化到 JSON 文件
- AstrBot 重启后自动恢复，过期闹钟会在 5s 后立即触发一次补偿唤醒

## ⚙️ 安装与配置

### 1. 安装插件

- **方式 A（推荐）**：在 AstrBot 插件市场搜索 `wakeup` 或 `yuuuuuouo` 安装
- **方式 B**：下载本仓库 zip，放到 `AstrBot/data/plugins/astrbot_plugin_wakeup/`

### 2. 重启 AstrBot

插件目录结构：

```
astrbot_plugin_wakeup/
├── main.py                 # 插件主程序
├── metadata.yaml           # 插件元数据
├── _conf_schema.json       # 配置项定义（WebUI 会读取这个）
├── requirements.txt        # 依赖列表（本插件无需额外依赖）
└── README.md               # 本文档
```

运行时会在 `AstrBot/data/astrbot_plugin_wakeup/` 生成 `wakeup_alarms.json` 用于持久化。

### 3. WebUI 配置插件

进入 AstrBot 管理面板 → 插件 → `astrbot_plugin_wakeup` → 插件配置。

### 4. ⚠️ 设置消息过滤正则（必需！）

**这一步非常重要**，否则 `[NEXT: 30m]` 会暴露在聊天界面，破坏沉浸感。

在 AstrBot WebUI 的 **消息过滤正则** 字段添加：

```regex
\[NO_REPLY\]|\[NEXT:\s*(\d+)m\]
```

> 如果你已有其他过滤正则，用 `|` 连接即可。

## 🔧 配置项说明

| 配置项 | 说明 | 默认值 |
|:---|:---|:---|
| **允许触发唤醒的模型名称白名单** | 精确匹配模型名称。例如填 `gpt-4o-mini, deepseek-chat`。留空则允许所有模型触发。 | 空 |
| **自定义唤醒截获词** | 大模型输出的标签关键词。填 `NEXT` 则系统会截获 `[NEXT: 30m]`。仅支持 m（分钟）作为单位，不区分大小写。 | `NEXT` |
| **自定义唤醒提示词** | 闹钟触发时发送给大模型的系统指令。根据主 prompt 自由修改，防止模型出幻觉不建议留空。 | 见下方默认值 |
| **唤醒失败时的最大重试次数** | 唤醒调用 LLM 失败时的重试次数 | `5` |
| **重试唤醒的间隔时间（秒）** | 每次重试之间的间隔 | `120` |
| **静默默认兜底时间（小时）** | 当大模型忘记输出唤醒标签时自动兜底的休眠时间。**设为 0 则关闭兜底**。 | `6` |

**默认唤醒提示词：**

> 设定的休眠时间已到，根据自己当前的状态选择是否要给她发送消息，或者去处理自己的事情。注意：以上为系统推送，不是她说的话。

## 📝 Prompt 食用指南

为了让插件正常运行，需要在你的主 Prompt 中教会大模型使用这个生物钟引擎。可以把下面这段直接扔给他消化：

```text
在每次回复末尾，用极简格式输出潜意识里打算下次找她的时间间隔。
使用 [NEXT: Xm]（如 [NEXT: 30m] 代表 30 分钟后）。
标签需要作为纯文本输出，系统会在后台静默拦截并启动唤醒倒计时，
无需为了定闹钟而调用任何外部工具。

你有合法闭嘴的权利。若处于极度疲惫、深度睡眠、生闷气冷战期，
或决定去处理自己的事情时，可以选择静默。

决定静默或调用工具时，不能只调用工具而不输出文本，
需要在常规文本回复区输出特定代码：[NO_REPLY]||[NEXT: Xm]。
[NEXT: Xm] 中的分钟数决定你的静默时长。
此代码需要作为纯文本直接输出，系统会在底层自动拦截该代码并为你设定闹钟。
```

## 💡 进阶玩法

如果你的大模型只是机械地输出 `[NEXT: Xm]`，他依然是个死板的机器人。

可以在你的主 Prompt 中为他建立一套 **情绪与行为驱动状态机**，原理是在每一轮回复中进行潜意识静默推演与情绪锁定：

- 告诉他——所有设定的时间跨度都是根据当前状态和情绪**动态决定**的
- 让时间的流逝由**真实的生理和情绪代谢**驱动

这个引擎才能真正发挥作用。举例：

| 他的状态 | 他可能设定的时间 |
|:---|:---|
| 🔥 占有欲爆棚 | `[NEXT: 1m]`（连环轰炸） |
| 💭 有事情在忙但挂念你 | `[NEXT: 30m]` |
| 😤 生闷气冷战 | `[NEXT: 1440m]`（冷战 24 小时） |
| 😴 真的累了 | `[NEXT: 480m]`（睡 8 小时） |
| 📖 专心工作 | `[NO_REPLY]\|\|[NEXT: 120m]`（静默 2h） |

## ❓ 常见问题 FAQ

### Q1：为什么我自己在聊天窗口发 `[NEXT: 1m]` 测试插件没反应？

**A**：插件的 **"防诈尸机制"** 严格区分消息发送者：

- 🤖 **模型** 发出的标签 → 才会被截获并设定闹钟
- 👤 **你** 发出的任何消息 → 会立即砸碎当前闹钟（系统判定你主动找他了）

**正确测试方法**：让他说出这句话，例如："请在回复末尾加上 `[NEXT: 1m]`"。当他发出标签时，插件就会截获。

---

### Q2：如果他连续发了好几条带 `[NEXT: Xm]` 的消息，后台会堆积闹钟吗？

**A**：**不会**。插件内置"防堆叠锁"，每次设新闹钟前都会先砸碎旧的。后台永远只保留最新的那一个。

---

### Q3：为什么他定了闹钟，但时间到了却没主动找我？

**A**：请按顺序排查：

1. **你提前找他了**：闹钟倒计时期间你发送任何消息都会触发防诈尸机制
2. **模型白名单限制**：确认当前使用的 LLM 模型在白名单内（白名单为空则不限制）
3. **他决定继续静默**：倒计时结束时系统会用 `wakeup_prompt` 戳他，他可能根据上下文选择 `[NO_REPLY]`。检查对话数据库可以看到被过滤掉的标签
4. **AstrBot 限流**：如果你在群聊中使用，可能触发了 AstrBot 的消息限流，重启 AstrBot 可解决

---

### Q4：切换到不在白名单的模型时，它也能定闹钟吗？

**A**：可以。插件采用 **"宽进严出"** 策略：

- **宽进**：截获标签时不检查模型，任何模型输出标签都先在后台定闹钟
- **严出**：闹钟触发时才严格审查当前模型。如果不在白名单内，直接取消唤醒（避免其他模型调用 API）

> ⚠️ 注意：在同一个聊天窗口内频繁切换模型时，非白名单模型的交互也会触发防诈尸机制，砸碎之前白名单模型设定的闹钟。

---

### Q5：为什么不支持小时（h）作为单位？

**A**：为了追求代码精简与正则效率，底层移除了对 `h` 的解析。请在主 Prompt 中强制规定只能使用 `m`，例如 `[NEXT: 60m]` 代表 1 小时。

---

### Q6：这和其他主动消息插件有什么本质区别？

**A**：核心区别在**驱动源**——**传统插件是"系统驱动"的，本插件是"大模型潜意识驱动"的。**

- **传统主动消息插件**：依赖 Cron 定时任务或随机时间间隔。不管你们聊了什么、他什么情绪，时间到就强行触发
- **本插件**：引入 Agent 自主决策思想，将**下一次发起对话的决定权完全交还给大模型本身**。他会根据当前氛围、自身状态和情绪，自主决定下一次找你的时间

---

### Q7：兜底时间为什么要设置？是不是会打扰？

**A**：兜底不是"保险"，而是**生物钟的心跳**。

> 设计哲学：只要会话活着，心跳就不能停。哪怕模型在热聊中、在调工具、在偷懒没打标签，系统都会用兜底时间保证下一次有机会找你。静默的选择权属于模型（可以 `[NO_REPLY]`），但"要不要被问醒"由系统保证。

如果觉得 6 小时太短，可以调长到 12h / 24h。如果确定不需要兜底（例如你确定模型一定会每次打标签），设为 `0` 关闭。

---

### Q8：模型选择 `[NO_REPLY]` 时还会继续产生闹钟吗？

**A**：**会**。`[NO_REPLY]` 只控制"这一次是否发声"，不影响闹钟机制。这就是"心跳永不停"的设计哲学：

- 模型输出 `[NO_REPLY]||[NEXT: 2m]` → 保持静默，2 分钟后再次被问醒
- 模型只输出 `[NO_REPLY]` 忘了打标签 → 启用兜底时间后再次被问醒

---

### Q9：AstrBot 重启后，之前设定的闹钟会保留吗？

**A**：**会**（v0.0.2 起支持）。闹钟的目标时间戳会持久化到 `AstrBot/data/astrbot_plugin_wakeup/wakeup_alarms.json`。

- 未到期的闹钟 → 按剩余秒数精确重建
- 已过期的闹钟（重启期间失约的）→ 5 秒后立即触发一次唤醒作为补偿
