# astrbot_plugin_wakeup

> 隐式生物钟与主动唤醒引擎 — 让 AI 自己决定什么时候找你聊天

## 简介

AstrBot 插件。让 LLM 在回复末尾输出 `[NEXT: Xm]` 标签来自主规划下次发言时间。插件静默截获标签并设定倒计时，时间到后构造伪造消息注入 AstrBot 的完整消息处理流程，让 LLM 根据对话上下文和自身状态决定是否主动发起对话。

**核心优势**：唤醒时走完整的 AstrBot pipeline，人格加载、对话上下文、MCP 工具调用、前台日志显示全部正常工作，与用户真实发消息触发的流程完全一致。

## 环境要求

- AstrBot v4.22+
- **aiocqhttp 适配器**（OneBot v11 协议）
- NapCat / Lagrange 等 OneBot v11 实现

**注意**：本插件仅支持 aiocqhttp 适配器，不支持其他消息平台。

## 安装

### 方式一：插件市场

在 AstrBot 管理面板的插件市场中搜索 `wakeup` 安装。

### 方式二：手动安装

cd AstrBot/data/plugins
git clone https://github.com/yuuuuuouo/astrbot_plugin_wakeup.git

重启 AstrBot 即可。无额外依赖。

## 配置

在 AstrBot 管理面板 → 插件管理 → wakeup 中配置：

| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| bot_qq_id | string | 空 | 机器人 QQ 号。留空自动检测，建议填写 |
| trigger_keyword | string | NEXT | 截获的标签关键词 |
| wakeup_prompt | text | 内置 | 闹钟触发时注入的提示词 |
| default_silence_minutes | int | 360 | 未检测到标签时的兜底休眠（分钟） |
| allowed_models | list | 空 | 模型白名单，留空 = 全部模型 |

## 使用方法

### 1. 在人格 Prompt 中引导 LLM 输出标签

在你的人格 system prompt 中加入类似说明：

每次回复末尾请输出 [NEXT: Xm]，X 为分钟数，代表你计划多久后再主动和用户说话。
例如：[NEXT: 30m] 表示 30 分钟后主动发言。
如果觉得用户短时间内不会回复，可以设长一点，比如 [NEXT: 360m]。

### 2. 可用指令

| 指令 | 说明 |
|------|------|
| /wakeup | 查看当前活跃的闹钟状态 |
| /smash | 手动砸碎当前会话的闹钟 |

### 3. 砸碎规则

- 用户发送**普通消息**会自动砸碎旧闹钟（LLM 回复后设新闹钟，形成循环）
- 发送 `/` 开头的**指令**（如 `/new`、`/help`）**不会**砸碎闹钟
- 需要手动砸碎请用 `/smash`

## 工作原理

用户发消息 → LLM 回复 "你好 [NEXT: 30m]"
↓
插件截获标签，设 30 分钟闹钟
↓
（等待 30 分钟...）
↓
构造伪造消息，注入 AstrBot pipeline
↓
完整消息处理流程自动执行：
加载人格 → 注入上下文 → 调用 LLM
→ 执行工具调用 → 发送回复给用户
↓
LLM 回复带新的 [NEXT: Xm]
↓
设下一个闹钟，循环往复

## 闹钟持久化

闹钟信息保存在本地文件中，AstrBot 重启后自动恢复：
- 未过期的闹钟按剩余时间继续倒计时
- 已过期的闹钟在重启后 5 秒内触发

## 已知风险与注意事项

### ⚠️ 使用前必读

1. **唤醒提示词会出现在对话历史中**：唤醒时注入的提示词会作为一条"用户消息"出现在 AstrBot 后台的对话历史里。用户在 QQ 端不会看到，但在 AstrBot 管理面板的对话记录中可见。

2. **其他插件会处理伪造消息**：伪造消息走完整 pipeline，意味着你安装的所有其他插件也会处理这条消息。如果你有关键词触发类插件、审核插件等，唤醒提示词可能触发它们。建议检查是否有冲突。

3. **每次唤醒消耗一次 API 调用**：唤醒走完整 pipeline，等同于用户发了一条消息，会产生一次 LLM API 调用费用。如果 LLM 在唤醒时调用了 MCP 工具，还会产生额外的工具调用。请注意兜底休眠时间不要设太短，避免频繁调用。

4. **依赖 aiocqhttp 内部 API**：本插件通过调用 aiocqhttp 适配器的内部事件处理方法来注入消息。如果 AstrBot 或 aiocqhttp 库的内部 API 发生变更，可能导致注入失败。失败后插件会自动设兜底闹钟稍后重试，不会永久沉默。

5. **仅支持 aiocqhttp 适配器**：不支持官方 QQ、Telegram、微信等其他平台。

6. **独立会话模式**：群聊仅支持开启了独立会话（unique_session=true）的配置。

## 常见问题

**Q: 唤醒时 LLM 能调用工具吗？**
可以。走完整 pipeline，MCP 工具调用完全正常。

**Q: 前台日志能看到唤醒消息吗？**
可以。和用户真实发消息的日志完全一样。

**Q: 其他指令（如 /new）会打断闹钟吗？**
不会。只有普通消息和 /smash 指令会砸碎闹钟。

**Q: 注入失败怎么办？**
插件自动重试 3 次，全部失败后设兜底闹钟稍后再试。

**Q: bot_qq_id 不填会怎样？**
插件会在收到第一条消息时自动检测。但如果重启后闹钟恢复触发时还没收到过消息，注入会失败（会自动重试）。建议填写。
